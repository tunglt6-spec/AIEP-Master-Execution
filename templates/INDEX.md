# AIEP Templates

Reusable, directly-usable templates for AIEP v1.0. Copy a template, replace every
`{{PLACEHOLDER}}`, and remove the guidance comments before committing.

| Template | Purpose |
| --- | --- |
| [work-order.template.md](./work-order.template.md) | Work Order with the exact frontmatter AIEP parses (id, title, phase, reviewLevel, status, owner) and the required body sections. |
| [adr.template.md](./adr.template.md) | Architecture Decision Record — Status, Context, Decision, Consequences, Alternatives. |
| [rfc.template.md](./rfc.template.md) | Request for Comments — Summary, Motivation, Design, Drawbacks, Alternatives, Open questions. |
| [sop.template.md](./sop.template.md) | Standard Operating Procedure — Purpose, Scope, Roles, Procedure, Checklist, References. |
| [review-summary.template.md](./review-summary.template.md) | Mirrors the platform-generated review summary (reviewer status, findings by severity, disposition). |
| [claude-self-review.template.md](./claude-self-review.template.md) | Claude self-review checklist mirroring the generated self-review artifact. |

## Conventions

- Frontmatter is a controlled, flat YAML subset — no nested mappings. Quote any
  title containing a `:`.
- `reviewLevel` ∈ `L1 | L2 | L3 | L4`; `status` ∈ `backlog | planned | in-progress | in-review | done | blocked`.
- Findings severities: `CRITICAL, HIGH, MEDIUM, LOW, INFO` (CRITICAL/HIGH are blocking).
- Review-summary and self-review artifacts are normally generated by `aiep review`
  into `.aiep/artifacts/<WO-ID>/`; the templates here document their shape and are
  for manual dispositions or planning.

See also: [../docs/sop/INDEX.md](../docs/sop/INDEX.md) for the operating procedures.
